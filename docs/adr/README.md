# Architecture Decision Records

This directory is the canonical ADR path for MemPR.

ADRs record binding architecture, product-scope, security, lifecycle, and
maintenance decisions. Council files in `../council/` are deliberation evidence,
not canonical decisions.

## ADR Rules

- Keep ADRs narrow and falsifiable.
- Put volatile research, market framing, and milestone sequencing in the PRD.
- Link council passes as supporting evidence instead of copying them.
- Mark future decisions as planned, not accepted runtime behavior.
- Update or supersede an ADR when implementation invalidates its assumptions.

## Post-1.0 Scope-Change Backlog

R1-R11 are accepted for the local-first 1.0 boundary. Future ADRs should be
opened for concrete changes rather than extending Phase 7 alphabetically.

Follow-up hardening that stays within the local-first product boundary is now
tracked by proposed ADRs for live adapter compatibility, scanner configuration,
diagnostics retention, reviewer identity, and model-assisted classification.

Scope-change ADRs are now proposed for hosted service, retrieval ranking, vector
search, embeddings, knowledge graphs, and compliance/legal-retention
guarantees. These proposals are planning gates, not shipped behavior.
