# Code of Conduct

MemPR should be a useful, direct, and low-drama project to work on.

Be respectful. Assume good intent when it is reasonable. Critique ideas and code,
not people. Do not harass, threaten, insult, or deliberately exclude contributors.

Project maintainers may remove issues, comments, or pull requests that make the
project harder to participate in. Repeated or severe behavior can lead to a ban
from project spaces.

Report conduct concerns by opening a private security advisory or contacting the
maintainer through the repository owner profile.
