# Code of Conduct

MemPR is intended to be a useful, direct, and low-drama project to work on.
Everyone participating in the project is expected to help keep it that way.

## Our Standards

Examples of welcome behavior:

- Be respectful and constructive.
- Critique ideas, designs, and code instead of people.
- Assume good intent when it is reasonable.
- Make room for maintainers and contributors to ask clarifying questions.
- Keep discussion focused on the project and its users.

Examples of unacceptable behavior:

- Harassment, threats, insults, or personal attacks.
- Sexualized language or imagery.
- Deliberate intimidation, stalking, or sustained disruption.
- Publishing private information without permission.
- Repeatedly ignoring maintainer requests to stop a behavior.

## Enforcement

Maintainers may edit or remove comments, issues, pull requests, or other
contributions that violate this code of conduct. Severe or repeated violations
may result in a temporary or permanent ban from project spaces.

Enforcement decisions are made to protect the health of the project and the
people participating in it.

## Reporting

Report conduct concerns through GitHub private vulnerability reporting when
available, or by contacting the repository owner through their GitHub profile.

Reports should include enough context for maintainers to understand what
happened. Maintainers will handle reports as privately as practical.
